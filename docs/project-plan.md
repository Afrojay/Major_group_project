# Project Plan

## Development approach

The project follows an Agile-inspired iterative lifecycle. The team began with a broad ISL glossary idea, produced an early Semester 1 prototype, reviewed supervisor feedback, and then re-scoped the project into a Django-based organisation-specific glossary platform.

## Main phases

1. Initial project idea and research
2. Semester 1 proof-of-concept prototype
3. Supervisor feedback and project review
4. Re-scoping and technology evaluation
5. Django project setup
6. Core data model implementation
7. Public glossary pages
8. Admin/content management
9. Testing and evaluation
10. Final report and demonstration preparation

## Current Django implementation tasks

- Create Django project structure.
- Add glossary app.
- Implement Organisation, Category, SignEntry and FAQEntry models.
- Register models in Django admin.
- Build organisation, category, sign detail, search and quick-reference pages.
- Add sample data for College Computing and Retail Customer Service.
- Add functional tests.
- Prepare documentation for requirements, system design, accessibility and testing.

## Review points

The main review points are:

- Can the platform show more than one organisation using the same codebase?
- Can users search and browse signs within an organisation?
- Can common signs be marked as quick-reference signs?
- Can content be managed through Django admin?
- Is the project scope clear enough for the final report and demonstration?
