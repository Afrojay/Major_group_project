# Testing Plan

## Functional testing

The prototype should be tested against its core requirements.

Initial tests:

- Home page loads.
- Organisation page loads.
- Category page loads.
- Sign detail page loads.
- Search returns expected signs.
- Quick-reference page only shows signs marked as quick reference.
- Django admin can create organisations, categories and signs.

## Usability testing ideas

Possible user tasks:

1. Open the Retail Customer Service glossary.
2. Find the sign for Receipt.
3. Search for Help.
4. Open the quick-reference page.
5. Browse a category and open a sign detail page.

The test should record whether the user completed the task, any confusion, and comments about readability or navigation.

## Accessibility review checklist

- Are headings used in a logical order?
- Can links and form fields be reached using the keyboard?
- Is focus visible?
- Is link text meaningful?
- Does the layout work on a mobile-sized screen?
- Are descriptions written clearly?

## Evidence for report

Screenshots, test results, notes from user feedback and explanations of fixes should be saved for the Testing and Evaluation chapter.
