# Requirements

## Functional requirements

- Users can view available organisation-specific glossary instances.
- Users can open an organisation glossary using a readable URL slug.
- Users can browse categories within an organisation.
- Users can view signs in a selected category.
- Users can open a sign detail page with term, video URL, description and usage context.
- Users can search signs within the selected organisation.
- Users can view quick-reference signs for commonly needed terms.
- Admin users can add, edit and delete organisations, categories, signs and FAQ entries through Django admin.

## Non-functional requirements

- The interface should be clear and readable.
- Navigation should be simple enough for a prototype demonstration.
- Pages should use semantic HTML and meaningful link text.
- The layout should work on mobile and desktop screens.
- The system should be maintainable by using Django models, views, templates and admin.
- The prototype should avoid unnecessary technical complexity.

## Prototype limitations

- Sample sign video URLs are placeholders until verified ISL video sources are added.
- The prototype does not replace qualified ISL interpreters.
- The prototype is not a full ISL course or complete ISL dictionary.
