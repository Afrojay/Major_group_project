# Accessibility Approach

Accessibility is a key design requirement for this project. For the prototype, accessibility is treated as a practical set of design choices that make the glossary easier to use and understand.

## Current accessibility goals

- Use clear page headings and semantic HTML.
- Keep navigation simple and consistent.
- Use meaningful link text such as sign names and category names.
- Use readable typography and good spacing.
- Provide a mobile-friendly layout.
- Include a skip link for keyboard users.
- Use visible focus styles for links, buttons and inputs.
- Keep descriptions in plain English.
- Make sign video content central to each sign entry.

## Video content

Each sign entry includes a video URL field. During prototype development, placeholder video links may be used. For final evaluation, real sign content should be verified and referenced properly.

## Limitations

The prototype does not yet include a full accessibility audit, captions/transcripts for every video, user testing with Deaf/ISL users, or advanced personalisation features. These are suitable areas for further work.
